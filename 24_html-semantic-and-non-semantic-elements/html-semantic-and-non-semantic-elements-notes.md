<!-- +++++++++++++++ Semantic and Non-Semantic Elements +++++++++++++++ -->
1. Semantic Elements:
=> Semantic elements clearly describe their meaning to both the browser and the developer.
=> They are meaningful HTML tags that define the type of content they contain.
Examples: <img>, <table>, and <article> - Clearly defines its content.

<!-- +++++++++++++++ List of Semantic Elements +++++++++++++++ -->
<header>: Represents the top section of a webpage (logo, heading, intro)
<nav>: Used for navigation links or menus
<main>: Contains the main content of the webpage
<section>: Divides content into sections
<article>: Holds independent, self-contained content (blog, post, news)
<aside>: Contains side content like ads or related links
<footer>: Defines the footer (copyright, contact, links)
<mark>: Highlights text (CSS is usually preferred)
<figure>: Holds self-contained content like images or diagrams
<figcaption>: Adds a caption to a <figure>
<time>: Defines date or time
<details>: Used to hide/show extra content
<summary>: Clickable heading for <details>

2. Non-Semantic Elements:
=> Non-semantic elements do not describe their content.
=> They have no meaningful definition and act only as containers.
=> These tags do not tell what type of content they hold.
=> They are just containers.
Examples: <div> and <span> - Tells nothing about its content.
